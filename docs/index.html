About

Account Lite is a lightweight accounting software with recording financial transactions (no inventory management available), classification(ledgers), and summarization(trial balance). The data is stored in a human readable format (which is very similar to CSV).

First Entry

After you're done setting up, the screen will look like as follows

>

Here you have to enter the command j which is used for entering journal entries. It'll ask you the following questions

  1. DEBIT (Debit account/ledger name)
  2. CREDIT (Credit account/ledger name)
  3. AMOUNT (The amount of the transaction)
  4. NARRATION (And finally, the narration)

After answering the questions, it'll show Journal #1 Entered. ðŸŽ‰

Example
> j
DEBIT cash
CREDIT salary
AMOUNT 1000
NARRATION being salary received
Journal #1 Entered
>
Caveat
  • All the fields are mandatory to create a journal entry.
  • The current date is taken as the date of transactions.
  • Ledger/Account names cannot include ',' (or comma).
  • Ledger/Account cannot be the same for both Debit and Credit
  • Amount can only be an Integer

View Trial Balance

After you're first journal entry you may want to view the Trial Balance. To do so, enter the command t which is used for displaying the Trial Balance. It'll only ask you 'As On' date of the Trial Balance. 

> t 
AS ON : <DD/MM/YYYY> [04/06/2021]
  1. <DD/MM/YYYY>: The format for writing the date.
  2. [04/06/2021]: The default input (Used when if no date is enter). This will be the current date.

After you enter the date, it'll show you the Trial Balance. ðŸŽ‰

View Ledger

Let's see how the ledger works. Enter the command l which is used to display a Ledger. It'll ask you the following questions.

  1. FROM (Opening date of the ledger or Balance B/D date)
  2. TO (Closing date of the ledger or Balance C/D date)
  3. ACCOUNT NAME (Ledger/Account name)

The ledger FROM and TO date inputs have the same way of entering dates as that of the Trial Balance.

Example
> l
FROM : <DD/MM/YYYY> [28/05/2021]
TO : <DD/MM/YYYY> [04/06/2021]
ACCOUNT NAME : cash

Documentation

There are 7 commands at this point in the software. âš  All the commands are case sensitive

CommandApplication
:exit

Use this command at any point to exit. If you're in middle of an operation, let's say entering a journal, you can use this command to exit the operation.

If you use this command outside an operation, it'll close the software.
cls

Clear your screen.
j

Create a journal entry.

This command will ask for the following inputs

  1. DEBIT: Debit side ledger of the journal entry.
  2. CREDIT: Credit side ledger of the journal entry.
  3. AMOUNT: Transaction amount.
  4. NARRATION: Transaction narration/ other details

Possible Errors :

  • Invalid Ledger Name: Cannot include comma in ledger name.
  • Invalid Ledger Name: Ledger name cannot be empty.
  • Invalid Ledger Name: Debit and Credit side cannot have the same ledger name.
  • Invalid Amount: Not a number.
  • Invalid Amount: Number is too big.
  • Invalid Amount: Cannot be less than or equal to zero.
  • Invalid Narration: Narration cannot be empty.
  • Unknown Error: Something went wrong with the database file.
jd

View all the journal entries.
jdf

View all the journal entries for a period.

This command will ask for the following inputs

  1. FROM: Starting date of the period. (7 days ago is the default)
  2. TO: Ending date of the period. (Today's date is the default)

Possible Errors :

  • Invalid Date: Date format should be DD/MM/YYYY and valid.
  • Invalid Date: Ending date is before Starting date.
l

View a particular ledger for a period.

This command will ask for the following inputs

  1. FROM: Opening date of the ledger. (7 days ago is the default)
  2. TO: Closing date of the ledger. (Today's date is the default)
  3. ACCOUNT NAME: Name of the Ledger/Account.

Possible Errors :

  • Invalid Date: Date format should be DD/MM/YYYY and valid.
  • Invalid Date: Ending date is before Starting date.
  • Invalid Data: field cannot be empty.
t

View the Trial Balance for a period.

This command will ask for the following inputs

  1. AS ON: 'As On' of the trial balance. (Today's date is the default)

Possible Errors :

  • Invalid Date: Date format should be DD/MM/YYYY and valid.
